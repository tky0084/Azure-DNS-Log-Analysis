### ローカルネットワークゲートウェイ(Hub側)の作成
<details><summary>詳細を見る</summary><div>

Azure Portalで、HubのVirtual Network Gatewayを開き、「設定 | 構成」より、以下を控えておく

- 自律システム番号(ASN)
- BGPピアのIPアドレスを控えておく
- パブリックIPアドレス

<br>

Azure Portalで「Local network gateways」と検索し、上部の「作成」を押下する。

![alt text](<スクリーンショット 2026-05-03 105246.png>)

<br>

「基本」タブにて、以下の情報を入力する

| 項目 | 入力方法 |
|:---------|:---------|
| サブスクリプション | Hubのリソースグループが存在するサブスクリプション |
| リソース グループ | Hubのリソースグループが存在するリソース グループ |
| リージョン | Hubの仮想ネットワークと同じリージョン |
| 名前 | lgw-hub-prod-prod-network-01 |
| エンドポイント | IP アドレス |
| IP アドレス | HubのVPNGatewayのパブリックIPアドレス |
| アドレス空間 | 空欄 |

![alt text](<スクリーンショット 2026-05-03 110949.png>)

<br>

「詳細設定」タブにて、以下の情報を入力する

| 項目 | 入力方法 |
|:---------|:---------|
| BGP 設定の構成 | はい |
| 自律システム番号(ASN) | HubのVPNGatewayの自律システム番号(ASN)|
| BGP ピアの IP アドレス | HubのVPNGatewayのBGP ピアの IP アドレス |

![alt text](<スクリーンショット 2026-05-03 111004.png>)

<br>

「確認および作成」で問題なければ作成する

![alt text](<スクリーンショット 2026-05-03 111012.png>)

<br>

### VPNGateway(Hub側)とローカルネットワークゲートウェイ(オンプレミス側)を接続

<details><summary>詳細を見る</summary><div>

ローカルネットワークゲートウェイ(オンプレミス側)の「設定 | 接続」より、追加を押下

![alt text](<スクリーンショット 2026-05-03 111656.png>)

<br>

「基本」タブにて、以下の情報を入力する

| 項目 | 入力方法 |
|:---------|:---------|
| サブスクリプション | Hubのリソースグループが存在するサブスクリプション |
| リソース グループ | Hubのリソースグループが存在するリソース グループ |
| 接続の種類 | サイト対サイト(IPsec) |
| 名前 | con-Hub-prod-toOnprem-01 |
| リージョン | Hubの仮想ネットワークと同じリージョン |

![alt text](<スクリーンショット 2026-05-03 112104.png>)

「設定」タブにて、以下の情報を入力し、作成する

| 項目 | 入力方法 |
|:---------|:---------|
| 仮想ネットワークゲートウェイ | HubのVPNGatewayを選択 |
| ローカルネットワークゲートウェイ | オンプレミス側のローカルネットワークゲートウェイ |
| 認証方法 | 共有キー（PSK） |
| 共有キー（PSK） | 任意の共有キー（これはオンプレミス側の接続作成でも使用します） |
| BGP を 有効にする | チェックを入れる |

![alt text](<スクリーンショット 2026-05-03 112025.png>)

## HubとSpokeの接続

<details><summary>詳細を見る</summary><div>

作製したSpoke環境の仮想ネットワークを開き、「ピアリング」→「追加」を押下する

![alt text](<スクリーンショット 2026-05-03 114246.png>)

<br>

リモート仮想ネットワークの概要を以下の通り入力する<br>
※　相手先(Hub環境側)からのピアリング設定です。

| 項目 | 入力方法 |
| -- | -- |
| ピアリング リンクの名前 | ピアリング名<br>例：peer-hub-prod-toSpoke1-01 |
| サブスクリプション | 相手先(Hub環境)のサブスクリプション |
| 仮想ネットワーク | 相手先(Hub環境)の仮想ネットワーク名 |

リモート仮想ネットワークのピアリング設定を以下の通り入力する

| 項目 | 入力方法 |
| -- | -- |
| 'vnet-hub-prod-network-01' に 'vnet-spoke-prod-network-01' へのアクセスを許可する | 〇 |
| 'vnet-spoke-prod-network-01' からのトラフィック転送の受信を 'vnet-hub-prod-network-01' に許可する' | × |
| 'vnet-hub-prod-network-01' 内のゲートウェイまたはルート サーバーに 'vnet-spoke-prod-network-01' へのトラフィックの転送を許可する | 〇 |
| 'vnet-spoke-prod-network-01' のリモート ゲートウェイまたはルート サーバーを使用するために 'vnet-hub-prod-network-01' を有効にする | × |

![alt text](<スクリーンショット 2026-05-03 114352.png>)

<br>

ローカル仮想ネットワークの概要を以下の通り入力する<br>
※　Spoke環境から見た設定です

| 項目 | 入力方法 |
| -- | -- |
| ピアリング リンクの名前 | ピアリング名<br>例：peer-Spoke1-prod-toHub-01 |

<br>

ローカル仮想ネットワークのピアリングの設定を以下の通り入力する

| 項目 | 入力方法 |
| -- | -- |
| 'vnet-spoke1-prod-network-01' に 'vnet-hub-prod-network-01' へのアクセスを許可する | 〇 |
| 'vnet-hub-prod-network-01' からのトラフィック転送の受信を 'vnet-spoke-prod-network-01' に許可する | × |
| 'vnet-spoke-prod-network-01' 内のゲートウェイまたはルート サーバーに 'vnet-hub-prod-network-01' へのトラフィックの転送を許可する | × |
| 'vnet-hub-prod-network-01' のリモート ゲートウェイまたはルート サーバーを使用するために 'vnet-spoke-prod-network-01' を有効にする | 〇 |

![alt text](<スクリーンショット 2026-05-03 114420.png>)

## SpokeのVMからオンプレミスのVMに疎通確認

<details><summary>詳細を見る</summary><div>

### 【事前準備】各VMにてMicrosoft Defeder Firewallの無効化

<details><summary>詳細を見る</summary><div>

> **この手順はSpoke環境、オンプレミス環境のすべてのVMに実施してください**<br>
> 仮想マシンは、初期状態では仮想マシン内のMicrosoft Defeder Firewallでping(ICMP)がブロックされています<br>
> この手順でMicrosoft Defeder Firewallをオフにし、pingが通るように設定します。<br>
> なお、すべて無効化することは、セキュリティ上よくないため、実際の現場では、現場ごとの適切な設定値にしてください。

仮想マシンにBastion経由でRDP接続し、スタートに「security」と検索して「Windows Security」を押下する

![alt text](<スクリーンショット 2026-05-04 003536.png>)

<br>

左側の「Firewall & network protection」を押下し、各Firewallを順番に押下する

![alt text](<スクリーンショット 2026-05-04 003621.png>)

<br>

Microsoft Defeder Firewallの有効化にかかるトグルボタンを押下し、Offにする

![alt text](<スクリーンショット 2026-05-04 003641.png>)

</div></details>
