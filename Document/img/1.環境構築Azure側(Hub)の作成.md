AzureFirewallは、事前にAzureFirewallSubnetができていることが前提となります。<br>
これができていないと作成画面でエラーとなるため、必ず事前に作成しておいてください。

<details><summary>詳細を見る</summary><div>

Azure Portalで、「Firewalls」と検索する<br>
![alt text](<スクリーンショット 2026-04-11 114224.png>)<br><br>

![alt text](<スクリーンショット 2026-04-11 114308.png>)<br><br>

| 項目 | 入力方法 |
|:---------|:---------|
| サブスクリプション | Hubのリソースグループが存在するサブスクリプション |
| リソース グループ | Hubのネットワークが存在するリソースグループ |
| 名前 | Azure Firewallの名前 |
| リージョン | 仮想ネットワークと同じリージョン |
| 可用性ゾーン | なし<br※この項目がない場合がありますが、デフォルトでゾーン冗長構成となります |

| 仮想ネットワークの選択 | 既存のものを使用 |
| 仮想ネットワーク | Hubのネットワーク |
| パブリック IP アドレス | 新規作成し、パブリックIPアドレス名を入力する|
| Firewall Management NIC | チェックを外す |<br>

## Azure DNS Private Resolverの作成
<details><summary>詳細を見る</summary><div>
Azure DNS Private Resolverとは、Microsoftが提供する名前解決を実行してくれるサービスです。<br>
簡単に言うと、Azure内とオンプレミスのDNSをつなぐ「中継役（DNSフォワーダー）」として使うことができます。<br><br>
今回のように、企業によってはオンプレミス環境に独自のDNSサーバーを持っている場合があります。<br>
そのような場合、Azure内のリソースがオンプレミスのドメイン名を解決したいとき、オンプレミスのDNSサーバーへ問い合わせを転送（フォワード）できます。<br>

仮想ネットワーク内に「リゾルバ（名前解決の仕組み）」を作ることができ、そのために以下の2種類のエンドポイントを配置します。

| 名前 | 役割 |
|:---------|:---------|
| インバウンドエンドポイント | 仮想ネットワーク内（Spokeなど）からのDNSクエリを受け取る役割 |
| アウトバウンドエンドポイント | 受け取ったDNSクエリを、別のDNSサーバーへ転送する役割 |


### InboundEndpointサブネットの作成

Hubの仮想ネットワークを検索し、左ペイン「設定 | サブネット」から、「+ サブネット」を押下する<br>
![alt text](<スクリーンショット 2026-04-11 121631.png>)<br><br>

以下の通りのパラメータを設定してサブネットを追加する

| 項目 | 入力方法 |
|:---------|:---------|
| サブネットの目的 | Default |
| 名前 | InboundEndpointSubnet |
| IPv4 アドレスの範囲 | 10.0.0.0/23(作製したHubの仮想ネットワークのIPアドレス範囲) |
| 開始アドレス | 10.0.0.96 |
| サイズ | /28 |<br>

![alt text](<スクリーンショット 2026-04-26 223509.png>)

### DNS Private Resolverの作成
Azure Portalで、「DNS Private Resolver」を検索し、「+ 作成」を押下する

![alt text](<スクリーンショット 2026-04-26 221339.png>)

「基本」タブを以下の通り設定する

| 項目 | 入力方法 |
|:---------|:---------|
| サブスクリプション | Hubのリソースグループが存在するサブスクリプション |
| リソース グループ | Hubのリソースグループが存在するリソース グループ |
| 名前 | dnsrz-hub-prod-prod-network-01  |
| リージョン | Hubの仮想ネットワークと同じリージョン |
| SKU | VpnGw1AZ |<br>
![alt text](<スクリーンショット 2026-04-26 223907.png>)


「受信エンドポイント」タブにて、「エンドポイントの追加」を押下し、以下の通り設定して「保存」を押下する

| 項目 | 入力方法 |
|:---------|:---------|
| エンドポイント名 | InboundEndpoint(任意) |
| サブネット | InboundEndpointSubnet |
| IP アドレスの割り当て | 動的(サブネットの中で自動で割り当て) |<br>

![alt text](<スクリーンショット 2026-04-26 224057.png>)

エンドポイントが追加されたことを確認

![alt text](<スクリーンショット 2026-04-26 224546.png>)

「送信エンドポイント」タブにて、「エンドポイントの追加」を押下し、以下の通り設定して「保存」を押下する

| 項目 | 入力方法 |
|:---------|:---------|
| エンドポイント名 | OutboundEndpoint(任意) |
| サブネット | OutboundEndpointSubnet |
| IP アドレスの割り当て | 動的(サブネットの中で自動で割り当て) |<br>


![alt text](<スクリーンショット 2026-04-11 122959.png>)

「ルールセット」タブにて、以下の通り入力する
※「ルール」の項目は後で設定します。

| 項目 | 入力方法 |
|:---------|:---------|
| ルールセットを追加します | チェックを入れる |
| ルールセット名 | dnsfrs-hub-prod-network-01(任意) |
| エンドポイント | OutboundEndpoint |<br>

![alt text](<スクリーンショット 2026-04-26 225122.png>)
