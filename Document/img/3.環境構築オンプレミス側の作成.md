### 仮想マシンの作製

オンプレミス側において、ADの冗長を構成します。<u>この手順は2回繰り返してください。</u>

Azure Portalで「仮想マシン」を検索し、「作成」を押下する

![alt text](<スクリーンショット 2026-05-02 090753.png>)

<br>

基本タブにて、次の通りに入力する
| 項目 | 入力方法 |
| -- | -- |
| サブスクリプション | Onprem環境のリソースグループがあるサブスクリプション |
| リソース グループ | Onpremの仮想ネットワークがあるリソースグループ |
| 仮想マシン名 | 任意の仮想マシン名 |
| リージョン | (Asia Pacific) Japan East |
| 可用性オプション | インフラストラクチャ冗長は必要ありません |
| セキュリティの種類 | Standard |
| イメージ | Windows Server 2025 Datacenter: Azure Edition - x64 Gen2 |
| サイズ | デフォルト |
| ユーザー名 | 仮想マシンの管理者アカウント名 |
| パスワード | 管理者アカウントのパスワード |
| パブリック受信ポート | なし |

![alt text](<スクリーンショット 2026-05-02 090959.png>)

<br>

ネットワークタブにて、下記の通り入力する

| 項目 | 入力方法 |
| -- | -- |
| 仮想ネットワーク | Onpremの仮想ネットワーク |
| サブネット | Onpremの仮想ネットワーク内の作成するサブネット名 |
| パブリック IP | なし |
| NIC ネットワーク セキュリティ グループ | Basic |
| パブリック受信ポート | なし |

![alt text](<スクリーンショット 2026-05-02 091403.png>)

<br>

確認及び作成で、作成を押下する

<br>

**この手順で、もう一台仮想マシンを作成してください**

## 仮想ネットワークのDNSサーバーの設定

> **仮想ネットワークのDNS設定とは**<br>
> この設定により、<u>その仮想ネットワーク内にあるすべてのデバイスの名前解決を一括設定できます。</u><br>
> 今回は、名前解決をオンプレミス側のADサーバー(上記で作成したVM)へデフォルトへ向けます。<br>
> ADサーバーには、DNSの役割りを持たせているケースが一般的に多いため、おおまかにDNSサーバー = ADサーバーと考えて問題ありません。<br>
> ADのセカンダリサーバーがドメイン参加するために必要となります。<br>
> ※オンプレミスには、オンプレミスのDNSサーバーがあると想定してください。

<details><summary>詳細を見る</summary><div>

作製したOnprem環境の仮想ネットワークを開き、「設定」→「DNS」をを押下し、DNS サーバーの項目について、変更を押下する

![alt text](<スクリーンショット 2026-05-02 093459.png>)

<br>

DNS サーバーをカスタムに指定し、カスタムDNSサーバーに、作製した仮想マシン2台のIPアドレスを指定して保存する

![alt text](<スクリーンショット 2026-05-02 102004.png>)

<br>

保存後、作製したAD用仮想マシンを2台とも再起動する

![alt text](<スクリーンショット 2026-05-02 102104.png>)

<br>

仮想マシン内でipconfig /allコマンドを実行すると、DNSサーバーが更新されている

![alt text](<スクリーンショット 2026-05-02 102608.png>)

</div></details>

## VPN Gatewayの作成
### GatewaySubnetの作製

<details><summary>詳細を見る</summary><div>

Onpremの仮想ネットワークを検索し、左ペイン「設定 | サブネット」から、「+ サブネット」を押下する

![alt text](<スクリーンショット 2026-05-02 102956.png>)

<br>

サブネットの目的を、「Virtual Network Gateway」にセットする<br>
また、開始アドレスを調整する（上記のdefaultサブネットは10.0.255.0/27(10.0.0.0 - 10.0.0.31)だったので、10.0.0.32を指定）<br>
サイズはMicrosoftの推奨値/27を指定(詳細はこちらhttps://learn.microsoft.com/ja-jp/azure/vpn-gateway/vpn-gateway-about-vpn-gateway-settings#gwsub)

![alt text](<スクリーンショット 2026-05-02 103226.png>)

<br>

作製されたことを確認する

![alt text](<スクリーンショット 2026-05-02 103326.png>)

</div></details>

### VPN Gatewayの作成

<details><summary>詳細を見る</summary><div>

Azure Portalで「virtual network gateway」と検索する

![alt text](<スクリーンショット 2026-04-11 120122.png>)

<br>

作成を押下する

![alt text](<スクリーンショット 2026-04-11 120138.png>)

<br>

以下の情報を入力してデプロイする

| 項目 | 入力方法 |
|:---------|:---------|
| サブスクリプション | Onpremのリソースグループが存在するサブスクリプション |
| 名前 | VPN Gatewayの名前 |
| リージョン | Onpremの仮想ネットワークと同じリージョン |
| SKU | VpnGw1AZ |
| 世代 | Generation1 |
| 仮想ネットワーク | Onpremの仮想ネットワーク |
| サブネット | 既定で割り当てられます |
| パブリック IP アドレス | 新規作成 |
| パブリック IP アドレス名 | パブリック IP アドレスの名前を指定 |
| アクティブ/アクティブ モードの有効化 | 無効 |
| BGP の構成 | 有効 |
| 自律システム番号 (ASN) | ここでは65516とする |
