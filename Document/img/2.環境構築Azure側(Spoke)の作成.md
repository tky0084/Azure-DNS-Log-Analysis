### 仮想マシンの作製

Azure Portalで「仮想マシン」を検索し、「作成」を押下する

![alt text](<スクリーンショット 2026-04-13 084931.png>)

<br>

基本タブにて、次の通りに入力する

| 項目 | 入力方法 |
| -- | -- |
| サブスクリプション | Spoke環境のリソースグループがあるサブスクリプション |
| リソース グループ | Spokeの仮想ネットワークがあるリソースグループ |
| 仮想マシン名 | 任意の仮想マシン名 |
| リージョン | (Asia Pacific) Japan East |
| 可用性オプション | インフラストラクチャ冗長は必要ありません |
| セキュリティの種類 | Standard |
| イメージ | Windows Server 2025 Datacenter: Azure Edition - x64 Gen2 |
| サイズ | デフォルト |
| ユーザー名 | 仮想マシンの管理者アカウント名 |
| パスワード | 管理者アカウントのパスワード |
| パブリック受信ポート | なし |

![alt text](<スクリーンショット 2026-04-13 085158.png>)

<br>

ネットワークタブにて、下記の通り入力する

| 項目 | 入力方法 |
| -- | -- |
| 仮想ネットワーク | Spokeの仮想ネットワーク |
| サブネット | Spokeの仮想ネットワーク内の作成するサブネット名 |
| パブリック IP | なし |
| NIC ネットワーク セキュリティ グループ | Basic |
| パブリック受信ポート | なし |

![alt text](<スクリーンショット 2026-04-13 085235.png>)

<br>

確認及び作成で、作成を押下する

### ルートテーブルにデフォルトルート(HubのAzureFirewall)の作製
すべての通信をHubのAzureFirewallへ経由させるためのルートを作成します。

なぜAzureFirewallをネクストホップとするのか<br>
AzureFirewallは、許可した通信のみを通して、それ以外は通さない仕組みになっています。<br>
Hubで通信可否を中央集中的に管理する場合、Spoke環境の通信経路はAzureFirewallを経由させることで意図しない通信が発生しにくくなります。<br>

<details><summary>詳細を見る</summary><div>

作製したルートテーブルにて、「設定」→「ルート」より、「追加」を押下し、以下の通り作成する<br>
※　ネクストホップをHub環境のAzureFirewallに指定します

| 項目 | 入力方法 |
| -- | -- |
| ルート名 | default |
| 宛先の種類 | IP アドレス |
| ネクスト ホップの種類 | 仮想アプライアンス |
| Propagate gateway routes | Yes |
| ネクスト ホップ アドレス | Azure FirewallのプライベートIPアドレス |

![alt text](<スクリーンショット 2026-04-13 090721.png>)

<br>

### Spoke環境のサブネットにルートテーブルを関連付け
ルートテーブルは、作成するだけでは動作しません。サブネットに関連付けることでそのサブネット内のルートに反映されます。

作製したルートテーブルにて、「設定」→「サブネット」より、「関連付け」を押下し、Spoke環境のVMがあるサブネットを選択してOKする

![alt text](<スクリーンショット 2026-04-13 090853.png>)

## 仮想ネットワークのDNSサーバーの設定

> **仮想ネットワークのDNS設定とは**<br>
> この設定により、<u>その仮想ネットワーク内にあるすべてのデバイスの名前解決を一括設定できます。</u><br>
> 今回は、名前解決をオンプレミス側のADサーバー(上記で作成したVM)へデフォルトへ向けます。<br>
> ADサーバーには、DNSの役割りを持たせているケースが一般的に多いため、おおまかにDNSサーバー = ADサーバーと考えて問題ありません。<br>
> ADのセカンダリサーバーがドメイン参加するために必要となります。<br>
> ※オンプレミスには、オンプレミスのDNSサーバーがあると想定してください。

<details><summary>詳細を見る</summary><div>

作製したOnprem環境の仮想ネットワークを開き、「設定」→「DNS」をを押下し、DNS サーバーの項目について、変更を押下する

![alt text](<スクリーンショット 2026-05-02 093459.png>)

<br>

DNS サーバーをカスタムに指定し、カスタムDNSサーバーに、作製した仮想マシン2台のIPアドレスを指定して保存する

![alt text](<スクリーンショット 2026-05-02 102004.png>)

<br>

保存後、作製したAD用仮想マシンを2台とも再起動する

![alt text](<スクリーンショット 2026-05-02 102104.png>)

<br>

仮想マシン内でipconfig /allコマンドを実行すると、DNSサーバーが更新されている

![alt text](<スクリーンショット 2026-05-02 102608.png>)

</div></details>
